== Configure SAML v2 for Okta

This documentation will guide you in configuring SAML v2 IdP for Okta. In this case, FusionAuth will act as Service Provider (SP) to Okta (IdP). Functionally, the result will allow you to display a "Login with Okta" button on your FusionAuth login page and connect via SAML to Okta users/applications.

image:identity-providers/okta-saml/okta-saml-result.png[result-intro,width=1200]

== Prerequisites
* A FusionAuth instance running on a publicly accessible URL. You can spin up a link:/pricing[basic FusionAuth Cloud instance] or link:/docs/v1/tech/installation-guide/[install it on any server].
* An link:https://www.okta.com/free-trial/[Okta account] connected to a business email address.
** The link:https://help.okta.com/eu/en-us/Content/Topics/end-user/ov-overview.htm?cshid=csh-user-ov-overview[Okta Verify app] if your Okta account is not configured to send SMS messages for 2FA.

== Okta configuration

Login to Okta and navigate to the `Admin` panel

image::identity-providers/okta-saml/okta-saml-admin-button.png[okta admin button,width=1200]

Navigate to [breadcrumb]#Applications -> Applications# and click the [field]#Create App Integration# button.

image::identity-providers/okta-saml/okta-saml-create-app-integration.png[create app integration,width=1200]

Select [field]#SAML 2.0# and click `+Next+`.

image::identity-providers/okta-saml/okta-saml-samlv2.png[saml v2,width=1200]

Give the app a name and click `+Next+`.

image::identity-providers/okta-saml/okta-saml-name.png[name okta app,width=1200]

Provide values for the [field]#Single sign-on URL# and [field]#Audience URI (SP Entity ID)#. We will change these values later, but you need to enter something in order to obtain information that will allow you to get the correct values from FusionAuth. 

image::identity-providers/okta-saml/okta-saml-dummy-values.png[okta dummy values,width=1200]

Scroll down to [breadcrumb]#Attribute Statements# and add the `+user.Id+` and `+user.email+` values. You can choose any names, but record them because you will need to supply them to FusionAuth later.

image::identity-providers/okta-saml/okta-saml-attribute-statements.png[attribute statements,width=1200]

Click `+Next+`, select one of the radio buttons on the [breadcrumb]#Feedback# page, and click `+Finish+`. Scroll to the bottom of the following page and click the `+View SAML setup instructions+` button. You will need to provide the information on this page to FusionAuth, so keep it open in a separate tab or window. 

image::identity-providers/okta-saml/okta-saml-configuration-details.png[saml configuration details,width=1200]

Return to the previous window and navigate to the [breadcrumb]#Assignments# tab. Click `+Assign+` and then `+Assign to people+`.

image::identity-providers/okta-saml/okta-saml-assign-to-people.png[assign to people,width=1200]

Click the `+Assign+` button next to the user whom you will authenticate. 

image::identity-providers/okta-saml/okta-saml-assign-button.png[assign button,width=1200]

== FusionAuth configuration

From your FusionAuth dashboard, navigate to [breadcrumb]#Settings -> Key Master# and click the downward arrow next to `+Generate Elliptic+`. Then click `+Import Certificate+`

image::identity-providers/okta-saml/okta-saml-import-certificate.png[import certificate,width=1200]

Give this key a name, for example `+OKTA SAML+`. Then, copy the `+X.509+` certificate from the Okta `+SAML setup instructions+` page that you left open previously. Paste it into the [field]#certificate# box in the FusionAuth Key Master. 

image::identity-providers/okta-saml/okta-saml-paste-certificate.png[paste certificate,width=1200]

Navigate to [breadcrumb]#Settings -> Identity Providers# and add a new `+SAML v2+` identity provider.

image::identity-providers/okta-saml/okta-saml-new-idp.png[new identity provider,width=1200]

Provide a name, for example `+OKTA SAML+`. For the [field]#IdP endpoint#, paste the [field]#Identity Provider Single Sign-On URL# from the Okta `+SAML setup instructions+` page. For the verification key, select from the dropdown the name of the key you just created in the Key Master. Optionally, change the [field]#Button text# to `+Login with Okta+`. 

image::identity-providers/okta-saml/okta-saml-new-idp-settings.png[new idp settings,width=1200]

Make sure your application has the `+Enabled+` switch toggled. Then, navigate to the `+Options+` tab and supply the names that you chose for the `+user.Id+` and `+user.email+` attribute statements in Okta. 

image::identity-providers/okta-saml/okta-saml-claim-options.png[claim options,width=1200]

Hit `+Save+` and `+View+` the identity provider you just created. Scroll down to the [breadcrumb]#SAML v2 Integration details# section and record these values so you can supply them to Okta.

image::identity-providers/okta-saml/okta-saml-integration-details.png[integration details,width=1200]

If you receive a warning after saving the identity provider, navigate to [breadcrumb]#Settings -> System# and add your Okta URL to the [field]#Allowed origins# list.

image::identity-providers/okta-saml/okta-saml-cors-warning.png[cors warning,width=1200]

image::identity-providers/okta-saml/okta-saml-cors-settings.png[cors settings,width=1200]

Return to your Okta dashboard. Navigate to [breadcrumb]#Applications -> Your Application# and edit the `+SAML Settings+`.

image::identity-providers/okta-saml/okta-saml-edit-saml-settings.png[edit SAML settings,width=1200]

Click `+Next+` to get to the `+Configure SAML+` page. Replace the [field]#Single sign-on URL# with the [field]#Callback URL (ACS)# integration detail from FusionAuth. Replace the [field]#Audience URI (SP Entity ID)# with the [field]#Issuer# integration detail from FusionAuth. Click `+Next+` and `+Finish+` to save these settings.

image::identity-providers/okta-saml/okta-saml-settings-from-fusionauth.png[settings from fusionauth,width=1200]

== Testing the Login

To test, navigate to the applications page in FusionAuth. Click on the [breadcrumb]#View# icon (magnifying glass) next to the application you enabled Okta login on and copy the `OAuth IdP login URL` address. Navigate to this address. You should see a `Login with Okta` option on your app's sign-in page:

image::identity-providers/okta-saml/okta-saml-result.png[result,width=1200]

Click the `Login with Okta` button. Test logging in with the username and password, as well as the 2FA method selected when configuring your account on Okta.

If it is all set up correctly, you should be redirected back to your app, successfully logged in. The user will be added to FusionAuth, and you can examine the [breadcrumb]#Linked accounts# section of the user details screen to see that the Okta OIDC link was created.
